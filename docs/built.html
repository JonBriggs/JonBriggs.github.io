Math Placement System

Placing incoming students into the right math course is a high-stakes, time-consuming judgment call. Every applicant arrives with a different combination of transcripts, test scores, and teacher evaluations, and the people making placements have to hold all of it in their heads at once. I built a system that does the heavy lifting while keeping the reasoning transparent.

It works in two stages. First, a multi-agent pipeline reads each student's profile independently and distills it into a clean, structured summary — one agent per student, so the messy source material never has to be processed all at once. Then a forward-looking placement model determines what math each student is currently doing and recommends the next course in the sequence, producing a workbook with placements, confidence scores, and short synopses that a human can review and override.

The model is deliberately tunable: thresholds, the way different signals are weighted, and calibration against prior years all live in configuration rather than buried in code. That means the people who own the placement process — not just the person who wrote the software — can adjust it and see how the results change.

The goal was never to automate judgment away. It was to give the people exercising that judgment a clearer, faster, more consistent starting point.

Canvas Bookmarklet Tools

Canvas holds enormous amounts of information about how a class is actually running, but it rarely surfaces the questions teachers and administrators most want answered. These are small, focused tools — bookmarklets that run entirely in your own authenticated browser session, talk only to Canvas's own API, and render a tidy overlay on the page. Nothing is sent anywhere; there are no accounts, servers, or secrets involved.

Grading Delay by Section

Run from any Canvas assignment page, this shows how long grading actually took, broken down by section: the median, the 90th percentile (the "90% of work graded within X" number), the maximum, a histogram of timing buckets, and a count of anything still ungraded. It's the kind of feedback loop that helps a department see its own turnaround honestly.

Seating Chart Builder

A fast way to turn a Canvas roster into a workable seating chart — built with a couple of colleagues to solve an everyday classroom problem rather than a technical one.
